Legend/Selectors

Colors go from darker to lighter signifying the importance of sessions (according to me, at least): Cours > TD > TP @Where Cours = Amphi = A TD = Salle = S TP = Lab = L Who Section = Sec Groupe = G Modules BD: Bases de données RC: Réseaux de télécommunication SE: Systèmes d'exploitation 1 DAW: Développement d'applications Web GL: Génie logiciel 1 TG: Théorie des graphes AJEL: Aspects juridiques et économiques des logiciels * In case an invalid input is provided, all cells get highlighted/selected. * The filter function basically takes a CSS selector and adds/removes the 'wanted' class from cells and rows to realize its job.

Examples

When does anyone study 'TP SE'? "TP.SE" When does Groupe04 have sessions in labs? "Groupe4.Lab" or simply "G04.L" or just "G4.TP" Show all Groupe6 or Section2's sessions! Details

* It must be hosted on a (local) server since it uses the fetch API which doesn't work on the 'file:' protocol. * It uses latest JavaScript features like Array.prototype.flatMap so the code probably needs to be transpiled/polyfilled to support relatively older browsers. * Tested using Google Chrome v71. * Developed using Notepad++ and lighttpd. Updating * When a new timetable is published, only the 'edt.htm' needs to be replaced. * The current 'edt.htm' file is 'EDT_S4_21-2-2019.pdf' as exported by MS Office Word 2016 ("save as Web Page, Filtered"). Possible improvements * Instead of 'fetch'ing the htm file, allow the user to select it using input type="file". This makes updating the EDT easier and allows this page to be opened using the 'file' protocol. * The CSS :has() pseudo-selector didn't work. My workaround is like meh. Find an alternative to do something like this: .row:not(:has(> .wanted.cell)) {display: none;} Inspiration This script is inspired by a code that my friend Wanis wrote in Python. His code parses an XML file (that was exported from a PDF which our university published) to output a list of students, sorted by their grades.
