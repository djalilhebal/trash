What?

- In case an invalid input is provided, all cells get highlighted/selected. - The filter function basically takes a CSS selector and adds/removes the 'wanted' class from cells to realize its job. - When a new timetable is published, only the 'edt.htm' needs to be replaced. - The 'edt.htm' file is 'EDT_S4_21-2-2019.pdf' as exported by MS Office Word 2016 ("save as Web Page, Filtered"). - It must be hosted on a (local) server since it uses the 'fetch' API which doesn't work on the 'file:' protocol. - Tested using Google Chrome v71. Developed using Notepad++ and lighttpd. - It uses latest JavaScript features like Array.prototype.flatMap so the code probably needs to be transpiled to support relatively older browsers. - Inspired by an interesting script that my friend Wanis wrote, which manipulated an XML file that was exported from a PDF. - Possible improvement: Instead of 'fetch'ing the htm file, allow the user to select it using input type="file". This makes updating the EDT easy and allows this page to be loaded using the 'file' protocol. - Check edt++'s JavaScript source code to see what's missing and needs to be done (mainly adding days' names).
